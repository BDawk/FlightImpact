Drop this folder onto a Pi with Chromium kiosk and the screens render at native 240×280. Every in-session screen carries a consistent status bar with battery, session label, and Wi-Fi state — green when the API is live, amber AP while the Pi is hosting its own network, and red with a slash when the web app cannot reach the Pi. Components that are required for capture (camera, radar, Uno) raise a device-not-ready alert when any are missing.
